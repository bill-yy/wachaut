# Plan: Panel Admin para Wachaut

## Arquitectura general

```
wadmin.billytech.es (subdominio)
  ├── SvelteKit SPA (misma web app, ruta /admin)
  │   └── Login UI → guarda JWT en sessionStorage
  │   └── Dashboard con métricas en vivo (polling cada 5s)
  │   └── Gestión de usuarios (crear/revocar admins)
  │
  └── API en el servidor Fastify (rutas /api/admin/*)
      ├── POST /api/admin/login → verifica contra DB, devuelve JWT
      ├── GET /api/admin/metrics → salas, viewers, SFU health (en vivo)
      ├── GET /api/admin/rooms → detalle de cada sala activa
      ├── GET /api/admin/stats → histórico (salas, picos, países)
      ├── GET /api/admin/security → IPs conectadas, rate limits
      ├── POST /api/admin/maintenance → modo mantenimiento (toggle)
      ├── GET /api/admin/users → lista de usuarios admin
      ├── POST /api/admin/users → crear nuevo admin
      ├── DELETE /api/admin/users/:id → revocar acceso
      └── POST /api/admin/users/:id/password → cambiar contraseña
```

## Seguridad (capas múltiples)

1. **Usuarios en DB** — tabla `admin_users` con username, password_hash (scrypt + salt), role (admin/superadmin), created_at, last_login, active.
2. **JWT con expiración corta** (1 hora). Firmado con `ADMIN_JWT_SECRET` (env var).
3. **Middleware Fastify** que verifica el JWT en cada ruta `/api/admin/*` excepto `/login`. Sin token válido → 401.
4. **Rate limiting** en `/api/admin/login` (max 5 intentos / 10 min por IP).
5. **Bootstrap del primer admin** — al arrancar el server, si la tabla `admin_users` está vacía y `ADMIN_BOOTSTRAP_PASSWORD` está en el env, crea un usuario `admin` con esa contraseña. Se puede eliminar del env después.
6. **Subdominio aislado** — CORS restringido a `wadmin.billytech.es`.
7. **Auditoría** — tabla `admin_audit_log` registra cada login (usuario, IP, país, timestamp) y acciones sensibles.

## Piezas a implementar

### 1. Base de datos — SQLite + better-sqlite3
- **Archivo:** `apps/server/src/db.ts`
- **Esquema:** `admin_users`, `admin_audit_log`, `room_events`, `traffic_stats`
- **WAL mode** activado; volumen Docker para persistir entre deploys

### 2. Geo-IP — geoip-country
- Lookups locales sin API, integrado en `realIp()`

### 3. Auth — apps/server/src/admin/auth.ts
- `scrypt` + salt aleatorio, JWT con jsonwebtoken
- Bootstrap del primer usuario admin

### 4. API Admin — apps/server/src/admin/routes.ts
- Endpoints de datos (metrics, rooms, stats, security, maintenance)
- Endpoints de gestión de usuarios (listar, crear, revocar, cambiar password)
- preHandler que valida JWT en todo excepto `/login`

### 5. Traefik routing — infra/docker-compose.prod.yml
- `Host(`wadmin.billytech.es`) && PathPrefix(`/api`)` → server
- `Host(`wadmin.billytech.es`)` → web (SPA)

### 6. Frontend Admin — apps/web/src/routes/admin/
- Login, Dashboard (métricas en vivo + países + seguridad + mantenimiento), Users (gestión de admins)
- Polling 5s, logout limpia sessionStorage

### 7. Script bootstrap — apps/server/src/admin/setup.ts
- CLI para crear usuarios manualmente

## Fases
- **Fase A:** DB + Auth + API + gestión de usuarios (backend)
- **Fase B:** Frontend (login + dashboard + gestión usuarios)
- **Fase C:** Deploy (Traefik + DNS wadmin.billytech.es + env vars + bootstrap)
