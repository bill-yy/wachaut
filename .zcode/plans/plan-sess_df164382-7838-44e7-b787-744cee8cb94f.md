# Plan: Mejora de calidad escalonada (VP9+bitrate → Simulcast)

## Fase 1: VP9 + Bitrate (seguro, 1 deploy)

### 1.1 `apps/web/src/lib/sfu-client.ts` — Cambiar a VP9

En `produce()`, el bloque de video producer actual no especifica codec (usa VP8 por defecto). Cambiar a VP9 explícito:

```typescript
this.#producer = await this.#sendTransport.produce({
  track: videoTrack,
  appData: { mediaTag: 'screen-video' },
  codec: 'video/VP9',                    // ← NUEVO
  codecOptions: {
    videoGoogleStartBitrate: Math.max(startBitrate, 100),
    videoGoogleMaxBitrate: maxBitrate,
  },
});
```

**Crítico:** NO añadir `encodings` ni `scalabilityMode`. VP9 sin SVC funciona perfectamente. El error de antes (`S3T3_KEY`) era el scalabilityMode, no VP9.

### 1.2 `apps/web/src/routes/room/+page.svelte` — Subir bitrates

Los presets actuales son conservadores para VP8. VP9 es más eficiente, pero para texto denso (código, documentos) necesitamos más bits:

| Preset | Bitrate actual | Bitrate nuevo | Razón |
|--------|---------------|---------------|-------|
| `low` (720p/15fps) | 1.0 Mbps | 1.5 Mbps | 720p con texto necesita más |
| `normal` (1080p/30fps) | 2.5 Mbps | 4.0 Mbps | 1080p texto denso necesita 4+ |
| `high` (1080p/60fps) | 5.0 Mbps | 6.0 Mbps | 60fps gaming + texto |

Custom default: 2.5 Mbps → 4.0 Mbps.

### 1.3 Verificación
- Deploy → probar en incógnito → comparar nitidez de texto
- Confirmar que sigue funcionando (sin OperationError)

## Fase 2: Simulcast VP8 (después de verificar Fase 1)

### 2.1 `apps/web/src/lib/sfu-client.ts` — 3 encodings sin scalabilityMode

Reemplazar el bloque VP9 de Fase 1 con simulcast VP8 (máxima compatibilidad):

```typescript
// Simulcast: 3 capas independientes a distintas resoluciones.
// El SFU entrega la capa apropiada a cada viewer según su conexión.
this.#producer = await this.#sendTransport.produce({
  track: videoTrack,
  appData: { mediaTag: 'screen-video', simulcast: true },
  codecOptions: {
    videoGoogleStartBitrate: 500,
    videoGoogleMaxBitrate: maxBitrate,
  },
  encodings: [
    { rid: 'low',    maxBitrate: 500_000,   scaleResolutionDownBy: 4 }, // 480p
    { rid: 'mid',    maxBitrate: 1_500_000, scaleResolutionDownBy: 2 }, // 720p
    { rid: 'high',   maxBitrate: maxBitrate },                            // full res
  ],
});
```

**Sin `scalabilityMode`** — cada encoding es un stream VP8 independiente. Esto es lo que usan Google Meet, Jitsi, etc. Chrome lo soporta universalmente.

### 2.2 `apps/sfu/src/index.ts` — Limpiar referencias a SVC

- El handler `createConsumer()` ya llama `setPreferredLayers({ spatialLayer: 1, temporalLayer: 2 })` con try/catch. Esto funcionará automáticamente con simulcast (mediasoup mapea los rids a spatial layers). No necesita cambios.
- El handler `host:set-spatial-layer` usa `setMaxSpatialLayer` — con simulcast esto controla la capa máxima enviada por el producer. Funciona sin cambios.

### 2.3 Verificación
- Deploy SFU + Web → probar con host + 2 viewers
- Verificar que viewers con buena conexión reciben `high`, viewers con mala reciben `mid`/`low`
- Revisar logs del SFU: `setPreferredLayers` debe funcionar sin error
